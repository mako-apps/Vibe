# Vibe Settings production refactor — team plan

## Enhanced brief

Refactor the iOS Settings experience into a production-grade, platform-native flow: correct section ordering; a functional connection-management sheet; real persisted privacy controls; authenticated server-backed notification preferences; complete appearance settings with live previews; and authenticated logged-in device/session listing with safe revocation of other sessions. Preserve current behavior and user changes, follow existing iOS/web patterns, use secure defaults, avoid unrelated refactors, build all iOS changes, test server work, and do not launch the physical app without confirmation.

## Architecture and contracts

- `settings-api-v1` — owner: Claude; consumer: Grok. Authenticated JSON for privacy preferences, notification preferences, and device sessions, with a current-session marker and safe revocation.
- `settings-store-v1` — owner: Grok; consumer: Agy. Observable typed iOS state/actions for loading, persistence, errors, previews/application, connection state, and session revocation.
- Advisor status: unavailable because the runtime cancelled the advisor call; conservative fallback split used.

- Changes: added `GET /api/settings`, `PATCH /api/settings/privacy`, and `PATCH /api/settings/notifications`; privacy uses explicit stable keys and existing validated user fields; notification updates deep-merge and reject unknown/non-boolean data.
